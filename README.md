# Sarathi Cyberdefense — Predictive Cyber Defense and Autonomous Remediation

## Problem Statement
This project addresses **PS10: Using Generative AI for Attack Prediction and Remediation**. Our project, titled *“A Generative AI Framework for Predictive Cyber Defense and Autonomous Remediation in Banking Systems”*, is an AI-powered cybersecurity platform for banks that proactively predicts, prevents, and responds to cyber threats. It builds a live security knowledge graph, ML-based risk forecasting and priority-based ranking mechanism, autonomous remediation playbook generation, root cause analysis, and a red team simulation in a novel Digital Twin to safely simulate attacks and validate security changes.

## How to Run Locally

1. **Clone the repo:**
   ```bash
   git clone [repository-url]
   cd sarathi-cyberdefense
   ```

2. **Environment Configuration:**
   Copy `.env.example` to `.env` in the root folder and populate it with your credentials:
   ```env
   NEO4J_URI=neo4j+s://your-database-id.databases.neo4j.io
   NEO4J_USERNAME=neo4j
   NEO4J_PASSWORD=your-aura-password
   GEMINI_API_KEY=your-google-gemini-key
   ```

3. **Backend Setup (Data Seeding & Server):**
   ```bash
   cd backend
   pip install -r requirements.txt
   
   # Seed the Neo4j graph with synthetic banking infrastructure
   python demo_data.py
   
   # Start the FastAPI backend
   python run.py
   ```
   *The backend runs at http://localhost:8000. Check http://localhost:8000/docs for the API reference.*

4. **Frontend Setup (Dashboard & Digital Twin):**
   Open a new terminal session.
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
   *The dashboard will be available at http://localhost:5173.*

## Project Structure
• `/backend` — FastAPI server, API routes, and orchestration logic.
• `/backend/demo_data.py` — Synthetic banking infrastructure and telemetry seeder.
• `/backend/ml/` — Machine Learning layer (Random Forest risk scorer and NetworkX attack path analyzer).
• `/backend/genai/` — Gemini playbook and Root Cause Analysis (RCA) generators with fallback cascade logic.
• `/frontend` — React 18 frontend dashboard application.
• `/frontend/src/components/twin/` — Three.js immersive Cyber Digital Twin visualization.

## Dataset
All internal banking telemetry is 100% synthetic (generated by our team using `demo_data.py`) to comply with strict banking privacy regulations (DPDP Act 2023, RBI data localization). 

It simulates a complex banking infrastructure across 4 security zones (DMZ, Middleware, Core, Management) featuring 12 key assets (e.g., Core Vault DB, SWIFT Gateway). However, **external threat intelligence feeds are real**:
• NVD (National Vulnerability Database)
• EPSS (Exploit Prediction Scoring System)
• CISA KEV (Known Exploited Vulnerabilities)
• MITRE ATT&CK Framework

No real bank transaction or network packet data was used.

## Model Performance (on Synthetic Test Set)
**Hybrid Risk Scoring Engine (Random Forest + Deterministic Formula):**
• Evaluates CVSS, EPSS, KEV status, and Asset Criticality.
• Successfully stratifies simulated vulnerabilities into CRITICAL, HIGH, MEDIUM, and LOW tiers deterministically for accurate prioritization.

**Dijkstra Attack Path Analyzer (NetworkX):**
• Pathfinding latency: Sub-millisecond.
• Accurately predicts the top-5 lateral movement routes (up to 6 hops deep) weighting edges by inverse asset criticality to find the "path of least resistance" to crown jewels.

*Note: These models are trained on a curated POC-scale dataset (12 records) for demonstration purposes. Production performance on real bank data would require retraining on 10,000+ labeled records.*

## Known Limitations
• **Synthetic Telemetry:** Internal SIEM/EDR data is simulated (due to privacy laws); external feeds (NVD, EPSS) are 100% real.
• **Batch Ingestion:** Graph data loads on startup; production scaling requires a real-time event streaming pipeline (e.g., Kafka).
• **ML Training Scope:** The Random Forest model uses a POC-scale curated dataset; production requires 10,000+ labelled records.
• **No Authentication:** The React dashboard lacks Role-Based Access Control (RBAC) and MFA, which are mandatory for a production SOC.
• **Static 3D Visualisation:** The Digital Twin visualises Neo4j graph state but does not ingest live raw network packet captures (PCAP).
